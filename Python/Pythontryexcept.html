Python Try...Except

Exception handling allows you to handle errors gracefully and prevent your program from crashing.

Basic Try...Except

try:
    print(x)
except:
    print("An exception occurred")

Handle Specific Exceptions

try:
    print(x)
except NameError:
    print("Variable x is not defined")
except:
    print("Something else went wrong")

Try...Except...Else

try:
    print("Hello")
except:
    print("Something went wrong")
else:
    print("Nothing went wrong")

Try...Except...Finally

try:
    print(x)
except:
    print("Something went wrong")
finally:
    print("The 'try except' is finished")

Raise an Exception

# Raise an exception
x = -1

if x < 0:
    raise Exception("Sorry, no numbers below zero")

# Raise a specific exception
if x < 0:
    raise ValueError("Sorry, no numbers below zero")

Common Exception Types

  • NameError: Variable is not defined
  • TypeError: Wrong data type
  • ValueError: Wrong value
  • IndexError: List index out of range
  • KeyError: Dictionary key not found
  • FileNotFoundError: File does not exist
  • ZeroDivisionError: Division by zero

Handle Multiple Exceptions

try:
    x = int("not a number")
    y = 10 / 0
except (ValueError, ZeroDivisionError):
    print("ValueError or ZeroDivisionError occurred")
except:
    print("Some other error occurred")

Get Exception Information

try:
    print(x)
except Exception as e:
    print(f"An error occurred: {e}")
    print(f"Error type: {type(e).__name__}")

Custom Exceptions

class CustomError(Exception):
    pass

def check_value(x):
    if x < 0:
        raise CustomError("Value cannot be negative")
    return x

try:
    result = check_value(-5)
except CustomError as e:
    print(f"Custom error: {e}")

Exception in Functions

def divide_numbers(a, b):
    try:
        result = a / b
        return result
    except ZeroDivisionError:
        print("Cannot divide by zero")
        return None
    except TypeError:
        print("Both arguments must be numbers")
        return None

# Test the function
print(divide_numbers(10, 2))    # 5.0
print(divide_numbers(10, 0))    # Cannot divide by zero
print(divide_numbers(10, "2"))  # Both arguments must be numbers

Assert Statement

# Assert statement for debugging
x = 5
assert x > 0, "x must be positive"

# This will raise AssertionError if condition is False
x = -1
assert x > 0, "x must be positive"  # AssertionError: x must be positive

Best Practices

  • Be specific with exception types
  • Don't use bare except clauses
  • Log exceptions for debugging
  • Clean up resources in finally blocks
  • Don't suppress exceptions unless necessary
